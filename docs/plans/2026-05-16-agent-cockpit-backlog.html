⚙ Comment lire cette page
  • Phases = vagues d'implémentation, ordre topologique strict (P1 bootstrap → P6 cockpit vNext).
  • Decision refs sur chaque tâche = liens directs vers les Decisions de l'interview (D1..D20).
  • Plumbing = tâches transverses (logging, error boundary, app identity…) qui n'existent dans aucune Decision mais sans lesquelles rien ne ship.
  • Demo paths = E2E tests qui rejouent les chaînes de vérification de bout-en-bout des Decisions clés.
  • vNext = explicitement marquées V2+ dans le bundle (cockpit grid, comment channel, apply hunks, libghostty FFI…). Restent dans le backlog comme placeholders.
