# agent-cockpit-term-sys

Raw FFI bindings to [libghostty](https://github.com/ghostty-org/ghostty).

## FFI confinement rule

This crate is the **only** place in the workspace allowed to declare
`extern "C"` items or hold `unsafe` blocks that touch libghostty. Every other
crate — including the safe wrapper `agent-cockpit-term` — must depend on the
typed Rust surface re-exported here and treat the library as if it were pure
Rust.

Concretely:

- Raw C bindings live here, generated by `build.rs` via `bindgen` (see P1-03).
- Linking to `libghostty.dylib` / `libghostty.so` is configured here (see
  P1-04 / P1-05).
- `unsafe fn` declarations from bindgen stay confined to this crate.
- Consumers import `agent-cockpit-term`, never `agent-cockpit-term-sys`
  directly, except for the safe wrapper itself.

This mirrors the convention used by `agent-cockpit-vcs` (no `unsafe`,
delegates risky work to its dependency `git2`).
